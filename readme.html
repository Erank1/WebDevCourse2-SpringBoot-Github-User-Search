Readme for ex3

In this exercise we asked to creates a GitHub search application similar to what we made last semester, but with SpringBoot and Thymeleaf template engine. We presenting history of searches from the database and have login page that only available when you logged out. To log in:
username: Admin
password: 1234

Client - Side

The client side is containing the following files:

  • Index.html - The main page and the only one in this app. content inserting the page dynamically. by Thymeleaf and AJAX (history)
  • error.html - the error page
  • login.html - the login page
  • headinclude fragment - to include head includes
  • navbar fragment - to include the navbar
  • GHSstyle.css - The custom Stylesheet of the app.
  • HistoryApp.js - The History application. it fetching from the server a json list of users and injecting it to the document.
  • mainApp.js - only to hide alerts after 5 seconds.

Server - Side

The server side containing the following servlets and classes:

    Beans

    • LoginBean - login form object and validation
    • LoginConstraint and LoginValidator are custom Validators for LoginBean fields.
    • DatabaseHandlerBean - to handle database conversations
    • JsonConverterBean - to handle Json conversion
    • LoginSessionBean - to handle login flag in session
    • URLHandlerBean - to handle url for search API

    Controllers:

    • CustomErrorController - Handle /error requests
    • DeleteController - Handle /delete requests
    • GHSearchController - Handle /do-search requests
    • HistoryController - Handle /history requests
    • LoginController - Handle /login requests
    • LogoutController - Handle /logout requests
    • MainController - Handle / requests

    Filters:

    • LoggingInterceptor - to block and redirect login if not logged in.

    Repo:

    • GHuser - the object entity bean we inserting to the DB
    • UserRepository - to send queries to the database.

    Application files:

    • Application
    • MyConfig
